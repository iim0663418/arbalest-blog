## 一個意外的交叉比對

強弩兵（Arbalest）是我花了一百多份 spec 慢慢長出來的個人 AI agent 系統。其中一個核心元件是 agent-memory——跑在 Cloudflare Workers + D1 + Vectorize + R2 上的記憶層。

幾週前我讀到 warmwater.dev 的八篇系列文，系統性地拆解了市面上的 agent memory 方案：Mem0、Letta、Zep 等等。分析框架很扎實，維度也夠多。我拿來跟自己的設計交叉比對，想看看哪裡重疊、哪裡分歧。

結果讓我意外。

不是分歧，是整塊缺席。我設計中幾個我認為最關鍵的維度，在那個分析框架裡完全不存在：

- **仿生睡眠週期**——NREM/REM 循環驅動的記憶整合與修剪
- **Active Inference 驅動的不確定性管理**——不是被動儲存，是主動預測與校正
- **感知-行動迴路**——記憶不是資料庫查詢，是 perception-action loop 的一部分
- **三層運算分離**——L1 確定性邏輯 / L2 本地 AI / L3 雲端推理，各層有不同的記憶存取模式
- **結果信號回饋**——寫入不是終點，outcome signal 決定記憶的後續命運
- **記憶安全與注入防禦**——當記憶本身可以被攻擊，你需要什麼防線

這不是說 warmwater.dev 的分析有問題。他們分析的是現有系統實際做了什麼。而我列的這些，現有系統確實沒做。

## 這個系列要回答的問題

一句話：**面對相同的問題，我為什麼做了不同的選擇，這些選擇跑在 production 裡長什麼樣子？**

我不打算寫學術論文，也不打算做產品行銷。這比較像是工程師的工作筆記——記錄設計決策背後的脈絡、實作時踩過的坑、以及跑起來之後觀察到的行為。

有些決策回頭看是對的。有些我還不確定。有些可能是過度設計。但它們都在跑，都有真實的 trace 可以看。

## 為什麼現在寫

強弩兵的 agent-memory 已經穩定運行一段時間了。睡眠週期在跑，Active Inference 在修正預測，R2 歸檔在壓縮歷史。我有足夠的 production 數據可以講具體的故事，而不是只講理論。

同時，我發現 agent memory 這個領域的公開討論大多停留在「怎麼存」和「怎麼取」。很少人談記憶的生命週期、記憶的主動性、記憶的安全性。這些是我想補上的空缺。

## 系列路線圖

這個系列預計八篇：

1. **為什麼不用現成的**——Mem0/Letta/Zep 的限制，以及我需要什麼它們給不了的
2. **寫入策略**——不是所有東西都值得記住，寫入的判斷邏輯
3. **生命週期設計**——記憶從誕生到消亡的完整路徑
4. **R2 歸檔**——冷記憶的壓縮、儲存與喚醒
5. **仿生睡眠**——NREM/REM 週期如何驅動記憶整合
6. **Active Inference**——用自由能原理管理記憶的不確定性
7. **記憶安全**——當攻擊者試圖污染你的記憶
8. **演化地圖**——系統怎麼從 v0 長到現在，下一步往哪走

每篇都會附上實際的架構片段、設定、或 production 觀察。不會有虛構的範例。

開始吧。
